# Plan 08-02: Glossary Page UI

## Objective

Create the glossary page with search, filtering, and A-Z navigation.

## Dependencies

- Plan 08-01 (glossary data exists)

## Implementation Steps

### Step 1: Create Glossary Page Route

Create `src/routes/glossary/+page.svelte`:
- Hero section with title and search bar
- Filter bar: category tabs or dropdown
- A-Z letter navigation (horizontal strip)
- Concept list grouped by letter
- Click concept to expand or navigate to detail

### Step 2: Create Search Component

Create `src/lib/components/glossary/GlossarySearch.svelte`:
- Search input with icon
- Instant filtering as user types
- Clear button
- Show result count
- Keyboard navigation support

### Step 3: Create Filter Component

Create `src/lib/components/glossary/GlossaryFilter.svelte`:
- Category filter (pills/tabs): All, Foundations, Methods, Assumptions, etc.
- Optional: Lecture filter dropdown
- Active filter indicators
- Clear filters button

### Step 4: Create A-Z Navigation

Create `src/lib/components/glossary/AlphabetNav.svelte`:
- Horizontal strip of letters A-Z
- Highlight letters that have concepts
- Dim/disable letters with no concepts
- Click to scroll to that section
- Sticky positioning option

### Step 5: Create Concept List

Create `src/lib/components/glossary/ConceptList.svelte`:
- Props: concepts array, groupByLetter boolean
- Letter section headers when grouped
- Concept cards in a grid or list
- Empty state when no results

### Step 6: Wire Up Page

- Combine all components on glossary page
- Manage filter/search state
- Handle URL params for deep linking (e.g., /glossary?q=ate)

## Verification

- [ ] /glossary route works
- [ ] Search filters instantly
- [ ] Category filters work
- [ ] A-Z navigation scrolls to sections
- [ ] Mobile responsive
- [ ] Build passes
