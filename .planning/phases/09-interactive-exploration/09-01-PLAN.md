# Plan 09-01: Exploration Page & Simulation Engine

## Objective

Create the interactive exploration landing page and core simulation utilities that power all the what-if scenarios.

## Dependencies

- Phase 5 (Animation System) for GSAP animations
- Phase 4 (Visualization Engine) for diagrams

## Implementation Steps

### Step 1: Create Exploration Route

Create `src/routes/explore/+page.svelte`:
- Hero section: "Interactive Exploration" title
- Subtitle about hands-on learning through what-if scenarios
- Grid of scenario cards linking to each interactive tool
- 3Blue1Brown-inspired aesthetic (dark theme option, smooth visuals)

### Step 2: Create Simulation Utilities

Create `src/lib/explore/simulation.ts`:
- `generateData(n, params)` - generate synthetic treatment/control data
- `addConfounding(data, strength)` - introduce confounding bias
- `addSelectionBias(data, strength)` - introduce selection bias
- `calculateATE(data)` - compute average treatment effect
- `calculateNaiveEstimate(data)` - compute biased estimate
- Random number generators with seed support for reproducibility

### Step 3: Create Interactive Controls Component

Create `src/lib/components/explore/ParameterSlider.svelte`:
- Svelte 5 component with smooth GSAP animation on value change
- Props: label, min, max, value, step, onChange
- Visual indicator of current value
- Optional color coding (e.g., red for bias strength)

### Step 4: Create Results Display Component

Create `src/lib/components/explore/ResultsPanel.svelte`:
- Animated number displays (count up/down on change)
- Shows: True ATE, Estimated ATE, Bias
- Color coding: green when estimate is close, red when biased
- Confidence interval visualization

### Step 5: Create Explore Component Index

Create `src/lib/components/explore/index.ts`

### Step 6: Add Explore to Navigation

Update Header.svelte to add "Explore" link in desktop nav.
